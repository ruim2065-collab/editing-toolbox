# 字体截图知识库

用途：存放剪映 / Capcut 字体截图索引，供“剪辑接单百宝箱”的字体识别工具调用。

## 当前素材

- 原始素材目录：`E:\剪辑\成片放在这\第四节：工具-字体收集`
- 来源：
  - `Capcut`：206 张
  - `剪映`：830 张
- 总数：1036 张
- 格式：全部为 `.png`

## 文件说明

- `font-screenshot-index.csv`：人工可查看、可补充的索引表。
- `..\..\api\data\font-screenshot-index.json`：服务端识别接口读取的索引数据。

## 字段说明

- `id`：知识库编号。
- `source`：来源，当前为 `Capcut` 或 `剪映`。
- `fileName`：原始截图文件名。
- `relativePath`：相对原始素材目录的路径。
- `originalPath`：本机原始截图路径。
- `width` / `height`：截图尺寸。
- `sha256`：原图哈希，用于学生上传同一张截图时精确命中。
- `visibleFontName`：截图上可见的字体名，当前等待 OCR 或人工补全。
- `ocrStatus`：当前为 `pending`，表示还没有批量 OCR。
- `styleTags`：风格标签，后续可补如 `手写`、`粗黑`、`圆体`、`综艺`、`英文`。
- `jianyingSearchKeywords`：剪映内建议搜索词。
- `usageNotes`：适用场景备注。

## 当前识别进度

- 已处理截图：1036 / 1036。
- 已确认字体名：889 个。
- 待人工复核 / 模型未确认：147 个。
- 当前没有待处理项。

说明：已确认字体名会优先作为百宝箱识别结果；待复核项不会硬填假字体名，会继续走截图视觉识别和相似方向推荐。

## 当前识别方式

1. 学员上传图片后，服务端先计算图片哈希。
2. 如果哈希与索引表一致，并且该截图已经确认字体名，直接返回“自建字体库命中”和 `visibleFontName`，不依赖外部视觉接口。
3. 如果哈希命中但字体名尚未确认，继续调用视觉识别读取截图里的字体风格和可能字体名。
4. 如果没有命中原图，也会按普通截图识别，并从已入库字体名中补充相似候选。
